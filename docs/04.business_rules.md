# Business Rules

## Internal Start Date Calculation

Workload is not allocated based on the customer Need Date.

Instead, an Internal Start Date is calculated using predefined lead time rules.

Formula:

```text
Internal Start Date = Need Date - Lead Time Days
```

The Internal Start Date determines when workload is assigned within the planning horizon.

---

## Lead Time Rules

Lead times are defined for each Product Type and Production Stage combination.

Example:

```text
PT1 + ST1 = 30 Days

PT1 + ST2 = 15 Days

PT2 + ST1 = 45 Days
```

These rules are used to determine when manufacturing activities must begin.

---

## Combination Concept

The planning process is built around the concept of a Combination.

A Combination represents:

```text
Product Type
+
Production Stage
```

Examples:

```text
PT1 + ST1

PT1 + ST2

PT2 + ST1
```

The Combination is used by:

- Master Plan
- Lead Time Rules
- Routing Matrix

This structure centralizes business logic and reduces data duplication.

---

## Routing Matrix

The Routing Matrix defines the workload required for each Combination and Work Center.

Structure:

```text
Combination
+
Work Center
+
Manned Hours
+
Installed Hours
```

Example:

```text
PT1 + ST1

WC1

Manned Hours = 3

Installed Hours = 2
```

---

## Workload Generation

Workload is generated by combining:

- Customer Demand
- Lead Time Rules
- Routing Matrix

Calculation flow:

```text
Master Plan

↓

Internal Start Date

↓

Routing Matrix

↓

Workload Generation
```

The result is the required workload for each Work Center.

---

## Production Flow Assumptions

For the purposes of this project:

- All Product Types pass through all Production Stages.
- All Production Stages generate workload for all Work Centers.
- Routing Hours determine how much workload is generated at each Work Center.

These assumptions simplify the manufacturing flow while preserving the analytical challenge.

---

## Detailed Workload Rules

Detailed workload is generated at the following grain:

```text
Project
+
Combination
+
Work Center
+
Internal Start Date
```

This level is used to preserve workload traceability.

It allows users to identify:

- Which project generated the workload
- Which production stage generated the workload
- Which Work Center will be impacted
- When the workload should begin

---

## Monthly Workload Rules

Detailed workload is aggregated into monthly buckets.

Grain:

```text
Month
+
Project
+
Combination
+
Work Center
```

Measures:

- Required Manned Hours
- Required Installed Hours

This dataset is used for workload and capacity comparison.

---

## Manned Capacity Calculation

Manned Capacity represents available labor hours.

The calculation considers:

- Workforce Allocation
- Shift Assignment
- Working Days
- Breaks
- Efficiency Factors

Available capacity is reduced by:

- Lunch Time
- Paid Breaks
- Indirect Activities
- Vacation Reserve Factor
- Holidays

The result represents Effective Manned Capacity.

---

## Installed Capacity Calculation

Installed Capacity represents machine availability.

For simplification purposes:

```text
Installed Capacity = 72 Hours per Day
```

for all Work Centers.

Monthly Installed Capacity varies according to:

- Calendar Days
- Working Days
- Holidays

---

## Capacity Planning Horizon

All capacity calculations are performed using monthly granularity.

Planning Horizon:

```text
12 Months
```

The solution is designed for:

- Tactical Planning
- Strategic Planning

The solution is not intended for detailed shop-floor scheduling.

---

## Capacity Snapshot Rule

Historical workforce assumptions must be preserved.

Example:

```text
January

20 Operators
```

```text
February

25 Operators
```

Result:

```text
January remains 20

February forward uses 25
```

Snapshots ensure planning traceability and historical comparison.

---

## Capacity vs. Workload Analysis

Two independent utilization calculations are performed.

### Manned Utilization

```text
Required Manned Hours
/
Available Manned Capacity
```

---

### Installed Utilization

```text
Required Installed Hours
/
Available Installed Capacity
```

This separation allows labor and machine bottlenecks to be analyzed independently.

---

## Capacity Classification

### Overload

```text
Utilization > 100%
```

---

### Balanced

```text
Utilization within target range
```

---

### Underload

```text
Utilization below target range
```

Thresholds may be adjusted during dashboard development.
