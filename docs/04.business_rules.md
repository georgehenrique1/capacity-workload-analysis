# Business Rules

## Internal Start Date Calculation

Workload is not allocated based on customer Need Date.

Instead, an Internal Start Date is calculated using predefined lead times.

Formula:

Internal Start Date = Need Date - Lead Time Days

---

## Lead Time Rules

Lead Time Rules are defined by:

- Product Type
- Production Stage

Example:

Product Type 1
Production Stage 1
Lead Time = 30 Days

---

## Routing Matrix Rules

Workload generation is driven by a routing matrix.

The routing matrix defines:

- Product Type
- Production Stage
- Work Center
- Manned Hours
- Installed Hours

Example:

PT1
ST1
WC1

Manned Hours = 3

Installed Hours = 2

---

## Workload Generation

Workload is generated by combining:

- Master Plan
- Internal Start Date
- Routing Matrix

Each project generates workload requirements for all applicable Work Centers.

---

## Production Flow Assumption

For the purpose of this project:

- All Product Types pass through all Production Stages.
- All Production Stages impact all Work Centers.
- Routing hours determine the workload generated at each Work Center.

This assumption simplifies the manufacturing network while preserving analytical complexity.

---

## Manned Capacity Calculation

Manned Capacity is calculated from:

- Operator allocation
- Work Center assignment
- Shift structure
- Working days

Available capacity is reduced by:

- Lunch time
- Paid breaks
- Indirect activities
- Vacation reserve factor (1/12)
- Holidays

The result represents Effective Manned Capacity.

---

## Installed Capacity Calculation

Installed Capacity represents the theoretical machine availability for each Work Center.

For this project:

Installed Capacity = 72 Hours per Day

Monthly capacity varies only according to:

- Number of calendar days
- Number of holidays
- Working day availability

---

## Capacity Snapshot Rule

A monthly snapshot of workforce allocation must be preserved.

Example:

January Snapshot:
20 Operators

February Snapshot:
25 Operators

January historical values remain unchanged even after future updates.

This rule ensures planning traceability and historical analysis capability.

---

## Capacity vs Workload Analysis

Two independent utilization calculations will be performed.

### Manned Utilization

Required Manned Hours
/
Available Manned Capacity

---

### Installed Utilization

Required Installed Hours
/
Available Installed Capacity

---

## Capacity Classification

Overload:
Utilization > 100%

Balanced:
Utilization between target thresholds

Underload:
Utilization below target thresholds

Threshold values may be adjusted during dashboard development.
